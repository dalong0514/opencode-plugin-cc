# Export Records

Export conversation JSONL files to `<项目>/chats/record/` as readable Markdown. The exporter keeps only substantive user and assistant text; it omits tool calls, tool results, reasoning, system prompts, and injected environment context.

## Usage

Determine the date filter and platform, then run the bundled script. Resolve the script relative to this skill directory.

```bash
python "scripts/export_records.py" --project "<CWD>" --filter <FILTER> --platform <PLATFORM>
```

Add `--force` when the user requests overwrite. Add `--include-subagents` only when the user explicitly asks for Codex delegated-agent conversations too.

| User intent | `--filter` |
| --- | --- |
| All conversations | `all` |
| Today's conversations | `today` |
| Specific date | `YYYY-MM-DD` |

| Execution environment / user scope | `--platform` |
| --- | --- |
| Running in Claude Code, or user explicitly asks for Claude Code records | `claude` |
| Running in Codex, or user explicitly asks for Codex records | `codex` |
| User explicitly asks to export both sources | `all` |

The script defaults to `claude` only for backwards compatibility with previous direct invocations. Do not rely on automatic filesystem detection to choose the platform: both tools may be installed on one machine, so the current execution environment or the user's wording is the reliable source of intent.

## Codex session scope

Codex stores rollouts in `~/.codex/sessions/YYYY/MM/DD/`. The exporter matches the `cwd` held in each session's metadata, rather than guessing from a directory name. By default, it exports only sessions marked `thread_source=user`, which prevents a multi-agent task from becoming dozens of noisy partial transcripts. If an older Codex version does not write that marker, it falls back to matching sessions and reports this in its output.

## Workflow

1. Determine `all`, `today`, or a `YYYY-MM-DD` date.
2. Choose the platform from the current environment or an explicit user request.
3. Run the script, adding `--force` or `--include-subagents` only when requested.
4. Read the script output and report the count, filenames, and any errors.

## Output format

Each conversation is saved as `YYYYMMDD-HHMMSS-{session_id8}.md` and includes:

- a title and metadata block (platform, session ID, project, creation time);
- `## 👤 User [HH:MM:SS]` and `## 🤖 Assistant [HH:MM:SS]` turns;
- `---` separators between turns.
